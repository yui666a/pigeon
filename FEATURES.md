# Pigeon 機能一覧

3ペイン構成（左: 案件ツリー / 中央: スレッド一覧 / 右: メール本文）を軸に、以下の機能を提供します。

## 📥 受信・同期

- **複数アカウント対応** — Gmail（OAuth 2.0）と任意のIMAPサーバー（PLAIN/LOGIN認証）を併用可能
- **初回一括同期** — デフォルトで直近5,000件を100件単位のバッチで取得。進捗表示付きで、途中終了しても次回続きから再開
- **差分同期** — UIDベースで前回以降の新着のみ取得。重複はDBのUNIQUE制約で防止
- **リアルタイム同期（IMAP IDLE）** — アカウントごとに常時接続でプッシュ受信。切断時は指数バックオフ（30秒→最大10分）で自動再接続。IDLE非対応サーバーは15分間隔ポーリングにフォールバック
- **デスクトップ通知** — 新着取り込み時に件数を通知。サイドバー下部のトグルでON/OFF

## 🤖 AI案件分類

- **自動グルーピング** — 件名・送信者・本文冒頭をLLMに渡し、既存案件への割り当て / 新規案件の作成 / 未分類を判定
- **確信度による振り分け** — 高確信度は自動分類、中間は⚠マーク付き、低確信度は「未分類」へ
- **新規案件の提案** — AIが案件名・説明を提案し、ユーザーが承認/修正/却下（1件ずつ逐次確認）
- **フィードバックループ** — 手動修正の履歴（correction_log）を次回以降のプロンプトにfew-shot例として注入。使うほど精度が向上
- **LLMプロバイダ切り替え** — Ollama（ローカル・デフォルト）/ Claude API / Claude on Vertex AI / Gemini on Vertex AI。クラウド選択時は警告を表示
- **案件ディレクトリ連携** — 案件にローカルフォルダを紐付けると、ファイル一覧から生成したコンテキストを分類判断に利用（クラウドLLMへのファイル由来データ送信はルールで制御）

## 🧵 スレッド・案件ビュー

- **スレッド構築** — `In-Reply-To` / `References` ヘッダーで返信チェーンを構築（RFC 2822準拠）。ヘッダー欠落時は件名の正規化（Re:/Fwd:除去）でフォールバック
- **案件ツリー** — 案件ごとの未読バッジ、色分け、アーカイブ、名前変更、マージ
- **未分類一覧もスレッド単位** — 返信の往復が1行にまとまり、件数バッジを表示
- **ドラッグ&ドロップ分類** — スレッドを案件へドロップすると、スレッド内の全メールをまとめて分類（修正履歴として学習にも反映）

## 📤 送信

- **新規作成 / 返信 / 全員に返信 / 転送** — 返信時は宛先・件名（Re:重複防止）・引用本文を自動プリフィル。全員に返信は自分のアドレスを自動除外
- **スレッディングヘッダー自動付与** — `In-Reply-To` / `References` をRFC 5322準拠で導出し、相手側のメーラーでも会話が繋がる
- **Sentフォルダ保存** — Gmailはサーバー側の自動保存を利用（二重保存なし）、その他はIMAP APPENDで保存
- **認証** — SMTP over STARTTLS（587）/ Implicit TLS（465）、PLAIN・LOGIN・XOAUTH2対応
- 送信成功はトーストで通知。失敗時は入力内容を保持したままエラー表示

## ✉️ メール操作

- **既読/未読** — 開封で既読化（サーバーへ`\Seen`反映）。他クライアントで読んだ分も同期時に取り込み。未読は太字表示 + 案件バッジ
- **削除** — サーバーに`\Trash`フォルダ（SPECIAL-USE）があれば**ゴミ箱へ移動**（Gmailはロケールに関わらず`[Gmail]/ゴミ箱`を自動発見）、無ければ完全削除。確認ダイアログ付き
- **アーカイブ** — INBOXから外して案件・検索には残す（Gmailはラベル剥がし、その他はArchiveフォルダへ移動）。削除とアーカイブは**メール1件ごとに選択可能**
- **アーカイブ解除** — 案件ビュー・検索から「アーカイブ解除」でINBOX表示に戻す
- **添付ファイル** — 一覧表示とダウンロード（オンデマンド取得 + ローカルキャッシュ。2回目以降はサーバーに接続しない）

## 🔍 検索

- **全文検索** — SQLite FTS5（trigram）で件名・本文・送信者・宛先を横断検索
- **案件横断** — 検索結果に所属案件を表示。ヒット箇所のスニペット付き

## ⌨️ キーボードショートカット

| キー | 操作 |
|------|------|
| `n` | 新規メール作成 |
| `r` / `a` / `f` | 返信 / 全員に返信 / 転送 |
| `e` | アーカイブ |
| `j` / `k` | 次 / 前のメール（スレッド内） |
| `/` | 検索にフォーカス |
| `Esc` | 作成モーダルを閉じる |

テキスト入力中・モーダル表示中は誤発火しません。

## 🔒 セキュリティ・プライバシー

- パスワード・OAuthトークン・APIキーは暗号化ストア（Stronghold）に保管。SQLiteに平文保存しない
- LLMへ送るのは件名・送信者・本文冒頭のみ。本文全文・添付・アドレス帳は送信しない
- デフォルトはOllama（ローカル）でメール内容がマシンの外に出ない。クラウドLLM選択時は明示的な警告を表示
- HTML本文はDOMPurifyでサニタイズして表示

## ⚠️ 既知の制限（v1）

- 送信はプレーンテキストのみ（リッチテキスト・添付ファイルの送信は将来対応）
- 下書きの保存・Drafts同期は未対応
- 自分が送信したメール（Sentフォルダ）はサーバー側の削除・アーカイブ・既読反映の対象外（ローカルのみ）
- アーカイブ解除はローカル表示のみ（サーバー側はアーカイブのまま）
- インライン画像（cid:）の本文内表示は未対応（添付一覧には表示）
- 通知は件数のみ（件名プレビューなし）

各制限の背景は `docs/superpowers/specs/` の該当設計書に記載しています。
