# Pigeon ペルソナ定義

機能の設計・優先度判断の際は、このペルソナのユースケースで価値が出るかを基準にすること。

## ペルソナ1: アスカ

| 項目 | 内容 |
|------|------|
| 名前 | アスカ |
| 属性 | 20代女性 |
| 職種 | エンタメ系の裏方スタッフ（担当領域: 照明） |
| 所属 | 舞台監督業を行う企業 |

### 業務環境

- 舞台監督は舞台に関する全てを把握している必要があるため、社内の慣習として**自分の業務領域に関係ないメールも全て届く**。
- 1つの舞台（公演）に対して、演出・飲食のケータリング・音楽・照明など、様々な取引先企業のスタッフとメールでやり取りが発生する。
- 結果として受信トレイには大量の異種混合メールが流れ込み、その中から自分の担当（照明）に関するものだけを拾う必要がある。

### コアニーズ

**大量のメールの中から、自分が担当する「照明」に関するメールだけを瞬時に検索・確認したい。**

### 検索における課題（表記揺れ・語彙の揺れ）

取引先やスタッフによって同じ対象の呼び方が揺れるため、単純な文字列一致では検索漏れが発生する。

- **表記揺れ**: 同一企業が `SATO` / `サトー` / `佐藤` のようにアルファベット・カタカナ・漢字で表記される
- **語彙揺れ（同義語）**: 「プリンター」を「端末」「デバイス」と呼ぶなど、同じ物・概念に別の語が使われる
- ひらがな/カタカナ、全角/半角、大文字/小文字の揺れも日常的に発生する

### Pigeon が提供すべき価値

1. 案件（公演）単位でのメール自動グルーピングにより、公演をまたいだノイズを排除できる
2. 案件内でさらに「照明」のような担当領域の観点で絞り込める検索
3. 表記揺れ・語彙揺れに頑健な検索（文字列一致だけに依存しない）
