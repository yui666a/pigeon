# セキュリティポリシー

Pigeon はメールの認証情報・本文・案件資料といった機微なデータを扱うデスクトップアプリケーションである。本ドキュメントは、脆弱性の報告方法と、Pigeon が採用しているセキュリティ上の設計方針をまとめる。

## 脆弱性の報告

セキュリティ上の問題を見つけた場合は、**公開の Issue を作らず**、非公開で報告してほしい。

- GitHub の [Security Advisories](https://github.com/yui666a/pigeon/security/advisories/new)（Private vulnerability reporting）から報告する
- 上記が使えない場合は、リポジトリオーナー（`yui666a`）へ直接連絡する

報告には次を含めてもらえると調査が早い。

- 影響を受けるバージョン / コミット
- 再現手順、または PoC
- 想定される影響（情報漏洩・権限昇格など）

個人開発プロジェクトのため、対応にかかる時間は保証できないが、報告は必ず確認する。

## 対象範囲

- **対象**: `main` ブランチの最新コード。Pigeon は現在 v1 開発中（`0.1.x`）であり、公開リリース配布はまだ行っていない。
- **対象外**: 依存ライブラリ自体の脆弱性（各上流へ報告してほしい。ただし Pigeon 側で緩和できるものは歓迎）、ユーザーが明示的に許可したクラウド LLM へのデータ送信。

## Pigeon のセキュリティ設計方針

以下は Pigeon が設計として担保している事項である。詳細な意思決定は `docs/adr/` を参照。

### 機密情報の保管（→ ADR 0003）

- パスワード・OAuth トークン・API キー・サービスアカウント JSON といった秘密情報は、OS のセキュアストレージ（Stronghold / SecureStore）に暗号化して保管する。
- **SQLite に平文の秘密情報を保存しない**。DB に置くのはプロバイダ名・モデル名・エンドポイントなどの非機密設定に限る。

### クラウド LLM へのデータ送信境界（→ ADR 0002）

- デフォルトの LLM はローカルで動作する Ollama であり、この場合データはマシンの外に出ない。
- クラウド LLM（Claude 等）を選択した場合でも、送信するのは件名・送信者・本文冒頭 1000 文字・ユーザーが明示的に許可した案件コンテキストに限定する。本文全文・添付ファイル本体は送信しない。
- 送信可否の判定はフェイルクローズ（判定不能なら送らない）を採る。案件ディレクトリ由来のファイルデータは、ユーザーが明示的に許可したもののみ送信する。

### その他

- `.env` や資格情報を含むファイルはコミットしない（`.gitignore` に記載済み）。
- メール本文の HTML は DOMPurify でサニタイズしてから表示する（`MailBody.tsx` / `SearchResults.tsx`）。加えて Tauri の CSP を `default-src 'self'`（画像は `data:`/`blob:` のみ許可）に設定し、外部リソースの読み込みやスクリプト実行を抑止する。
